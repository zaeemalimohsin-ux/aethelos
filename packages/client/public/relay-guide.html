Run an AethelOS relay

A relay is a powerless mailbox. It holds no ledger, no keys, and no authority — only a short-lived buffer so offline devices can catch up. Anyone can run one; communities can use several at once.

After your relay is reachable on the internet (with wss:// for HTTPS sites), paste its URL under Identity → Advanced → network, or set it when starting a community under Advanced relay settings.

Quick start (Docker)

From the AethelOS repo on a VPS or home server:

docker compose up -d --build
# Relay listens on port 8787
# Health check: http://YOUR_HOST:8787/healthz

Put TLS in front (required for HTTPS clients)

Browsers on secure sites need wss://. Terminate TLS with Caddy or nginx: 

# Caddy example — automatic HTTPS
relay.example.org {
  reverse_proxy localhost:8787
}

Communities then use wss://relay.example.org.

Run from source

pnpm install
pnpm --filter @aethelos/core build
pnpm --filter @aethelos/relay build
node packages/relay/dist/index.js

Resilience

  • Run multiple independent relays in different places.
  • Add each URL to your community — members sync through all of them.
  • A bad relay cannot forge state; every device verifies signatures locally.
